Prim's Algorithm for Minimum Spanning Tree

Prim's Algorithm Description

Prim's algorithm is a greedy algorithm that finds a minimum spanning tree for a weighted connected graph. A minimum spanning tree is a subset of the edges that connects all the vertices in the graph with the least possible total edge weight.

        ALGORITHM Prim(G)
        // Prim’s algorithm to construct a minimum spanning tree
        // Input: A weighted connected graph G(V, E)
        // Output: ET, the set of edges composing the MST of G
        VT ← {v₀}
        ET ← Ø
        for i ← 1 to |V| - 1 do
          find a minimum weight edge e* = (v*, u*) along all the edges (v, u) such that
          v is in VT and u is in V – VT
          VT ← VT ∪ {u*}
          ET ← ET ∪ {e*}
        return ET

Time Complexity of Prim's Algorithm

The time complexity of Prim's algorithm depends on the data structure used to represent the graph and implement the priority queue.

Data Structure Time Complexity Explanation
Simple Array (No Priority Queue) O(V^2) Each selection of the minimum edge and update takes O(V) time, leading to O(V^2) for all vertices.
Binary Heap (Min-Heap) O((V + E) log V) Extracting the minimum edge and updating vertices takes O(log V) time, resulting in a more efficient algorithm, especially for sparse graphs.
Fibonacci Heap O(E + V log V) Decrease-key operations are more efficient with O(1) amortized time, and extracting the minimum edge takes O(log V), making it ideal for dense graphs.
